# 專案開發規範與注意事項 (DEVELOPMENT_GUIDELINES)

本文件記錄了本專案的所有開發注意事項。後續所有的功能調整、擴充與維護，都必須嚴格遵守以下開發規則：

---

## 1. 運行環境規範
* **Python 版本**：必須使用 **Python 3.12** 或以上版本。
* **依賴管理**：所有第三方套件都應記錄於 `requirements.txt` 中，並透過虛擬環境 (`venv`) 進行隔離。

---

## 2. 日誌 (Logging) 記錄規範
* **時區要求**：日誌中的時間戳記必須統一使用 **台灣時間（台北時區，`Asia/Taipei`，UTC+8）**。
* **充分性**：必須記錄應用程式的生命週期事件，包含：
  * 設定載入與初始化。
  * 永豐 API (`shioaji`) 登入嘗試及結果。
  * 行情訂閱事件與回呼（Callback）觸發。
  * 錯誤與異常狀況（如網路中斷、金鑰錯誤等）。
  * 程式關閉與資源釋放。
* **易讀性**：
  * 日誌格式應包含：`[時間] [層級] [模組名稱] - 訊息內容`。
  * 日誌同時輸出至終端機與檔案（如 `app.log`），以便後續追蹤。
* **日誌輪轉 (Log Rotation)**：
  * 必須採用每日輪轉的方式記錄日誌，每天產生一個新的日誌檔案（例如使用 `TimedRotatingFileHandler` 於每日午夜進行切換）。
  * 應設定保存期限限制（例如最多保留 30 天的日誌），避免佔用過多磁碟空間。

---

## 3. 程式碼註解規範
* **語言要求**：程式碼中所有的註解與說明文字必須使用**繁體中文**。
* **充足性**：
  * 對於永豐 API (`shioaji`) 的特殊調用與流程，必須加上詳細說明。
  * 複雜的邏輯控制（如非同步回呼、線程安全、終端畫面更新控制）須標註設計用意。
  * 類別 (Class) 與函數 (Function) 均須編寫中文 Docstring，標明輸入參數與回傳值。

---

## 4. 終端機 UI 呈現規範
* **原地更新**：即時的期貨價格和大盤指數價格**必須在終端機的同一個位置更新呈現**，不可持續向下滾屏。
* **可讀性**：畫面應具備良好的排版，區分期貨與大盤的價格、漲跌幅、成交量，並顯示當前連線狀態與最後更新時間。
* **防閃爍與跳動設計**：
  * 為避免終端機更新時畫面頻繁重繪而產生閃爍與跳動，必須使用**獨立終端緩衝區 (全螢幕模式，如 Rich Live 的 `screen=True`)**。
  * 應採用**事件驅動/變更檢測渲染機制**（例如使用資料版本號 `version`），僅在資料有實際變動或當前秒數改變時才重繪畫面，不可於循環中無條件重繪。
* **容錯性 (Mock Fallback)**：
  * 當 API 金鑰未設定或登入失敗時，程式不能崩潰，應記錄 Error/Warning Log 後自動切換至 **Mock Fallback (模擬演示模式)**，產生模擬價格走勢，讓使用者可以直接在終端上看見效果。

---

## 5. 操作說明與文件規範
* **說明手冊要求**：所有新增功能、指令或運行參數變更，都必須同步記錄於 `README.md` 檔中。
* **內容完整性**：說明文件必須明確記載以下內容：
  * 環境準備步驟（如 Python 3.12、`requirements.txt` 安裝與 `venv` 虛擬環境建立）。
  * 憑證設定指引（`.env` 金鑰配置與 `SHIOAJI_SIMULATION` 模擬開關）。
  * 所有指令參數的教學（標準監控、極簡監控 `-m`、帳戶監控 `-a` 與單次查詢 `-q`）與執行範例。
  * 系統提示與常見錯誤處理（如 406 錯誤的成因與解決辦法）。
