# 台股期貨與大盤即時監控系統 (SPF API Monitor)

這是一個基於 Python 3.12 開發的終端機（CLI）即時報價監控與帳戶部位管理系統。本系統採用**永豐金證券 Shioaji API**，能實時獲取台股期貨（`TXFR1`）與加權大盤指數（`TSE001`）報價，並提供豐富的個人期貨帳戶可用餘額、風險指標及未平倉部位的追蹤呈現。

---

## 📋 系統特點
1. **同位置原地更新**：使用 `Rich Live` 技術，畫面更新無閃爍且不滾屏，始終在終端的同一位置動態排版。
2. **多重執行模式**：支援「綜合監控」、「極簡行情」、「帳戶專屬監控」與「靜態單次查詢」等多樣模式。
3. **安全 Mock 備援**：未設定 API 金鑰或伺服器連線失敗時，自動降級切換至 `Mock 模擬演示模式`，產生隨機跳動數據以供快速測試。
4. **全繁體中文註解**：程式碼編寫詳細的傳統中文說明，並嚴格遵循台灣時間時區（`Asia/Taipei`, UTC+8）與每日定時日誌輪轉。

## ⚙️ 環境準備與安裝

### 1. 建立虛擬環境與安裝依賴
本專案依賴 Python 3.12+。在專案目錄下執行以下指令以初始化虛擬環境並安裝所需套件：
```bash
# 建立虛擬環境 (名為 venv)
python3.12 -m venv venv

# 安裝依賴套件
venv/bin/pip install -r requirements.txt
```

### 2. 設定 API 金鑰憑證
請複製範本檔案 `.env.example` 並命名為 `.env`：
```bash
cp .env.example .env
```
並編輯 `.env` 填入您的永豐 API 金鑰與設定參數：
```ini
# 永豐 API 金鑰 (API Key)
SHIOAJI_API_KEY="您的永豐API_KEY"

# 永豐 API 密鑰 (Secret Key)
SHIOAJI_SECRET_KEY="您的永豐SECRET_KEY"

# 是否使用模擬交易伺服器 (True 代表模擬交易；False 代表真實帳戶實時連線)
SHIOAJI_SIMULATION=True
```
*註：若不設定金鑰或 `.env` 檔案不存在，程式會自動以「Mock 模擬模式」運行。*

## 🚀 運行指令教學與模式說明

### 1. 綜合即時監控模式 (預設模式)
同時訂閱大盤加權指數與期貨近月報價，並每 10 秒定期向伺服器拉取最新的帳戶剩餘保證金水位與持倉部位（可有效避免觸發 API 頻率限制）。
- **運行指令**：
  ```bash
  venv/bin/python3 main.py
  ```
- **主要畫面元素**：
  - **頂部 Header**：顯示系統運行模式與當前台灣時間。
  - **行情報價 Table**：大盤及期貨成交價、漲跌點數、漲跌幅、開高低盤與成交量。
  - **帳戶與持倉 Panel**：帳戶權益、可用保證金、期貨部位總損益、風險指標，以及個別部位的進場價與估算損益。
  - **底部日誌 Panel**：在不打亂介面的情況下，呈現最新 4 條系統日誌。

### 2. 極簡行情監控模式 (`-m` / `--minimal`)
僅在終端畫面上更新顯示單行的「時間、期貨價格與漲跌幅」，過濾掉所有表格、大盤指數與帳戶資訊。適合將終端機視窗縮到極小，放置在螢幕邊角監控。
- **運行指令**：
  ```bash
  venv/bin/python3 main.py -m
  # 或
  venv/bin/python3 main.py --minimal
  ```
- **輸出範例**：
  `[12:06:34] 台指期貨近月 (TXFR1): 44910 (+10 / +0.22%)` (會隨著價格變動原地閃爍更新)

### 3. 帳戶即時監控模式 (`-a` / `--account`)
如果您希望單純監看期貨帳戶與未平倉持倉的即時盈虧，而不希望加載大盤與期貨的行情走勢。此模式下**不會訂閱**任何 Tick 報價，有助於降低網路負載與 API 限制風險。
- **運行指令**：
  ```bash
  venv/bin/python3 main.py -a
  # 或
  venv/bin/python3 main.py --account
  ```
- **主要畫面元素**：僅包含帳戶保證金、風險指標、持倉部位表格與底部日誌，不含報價行情 Table。

### 4. 帳戶單次查詢模式 (`-q` / `--query`)
同步連線登入永豐 API，查詢一次您的期貨帳戶資金餘額與持倉部位，並以 Rich 格式在終端機印出精美的靜態表格與摘要後**立即退出程式**。此模式非常適合串接 Shell Script 或用於快速檢查。
- **運行指令**：
  ```bash
  venv/bin/python3 main.py -q
  # 或
  venv/bin/python3 main.py --query
  ```

### 5. 模擬環境功能交易測試與報告生成模式 (`-t` / `--test-report`)
向券商申請開通 API 正式下單權限前，通常需要提交一份在模擬環境（Simulation=True）下進行下單、改價、刪單的交易日誌測試報告。本專案已將此測試流程完全自動化。
- **運行指令**：
  ```bash
  venv/bin/python3 main.py -t
  # 或
  venv/bin/python3 main.py --test-report
  ```
- **自動化測試流程**：
  1. 連線至模擬交易環境。
  2. 載入並啟用您的 CA 憑證。
  3. 發送期貨買進限價 ROD 單 1 口（委託價格會防呆設為最新市價減 1000 點，確保不成交）。
  4. 發送改單要求（調升委託價格 100 點）。
  5. 發送撤單要求（撤銷該筆委託）。
  6. **產出測試報告**：在專案根目錄下自動生成符合永豐金格式要求的 `shioaji_simulation_test_report.md` Markdown 報告檔案。
- **注意**：依券商規定，模擬下單亦需要憑證。請確保您已在 `.env` 中設定 `SHIOAJI_CA_PATH` 與 `SHIOAJI_CA_PASSWORD`。若未設定憑證，測試器將自動進入「模擬日誌模式」產生報告範本，供您預覽。

## 🔍 常見問題與異常排除 (Error 406)

### ❓ 執行查詢時帳戶面板出現「查詢失敗 (錯誤碼 406)」
- **異常現象**：
  ```
  ╭──────────────────────── 💼 期貨帳戶保證金與持倉部位 ─────────────────────────╮
  │ ╭──────────────────────────────────────────────────────────────────────────╮ │
  │ │ ⚠️ 查詢失敗 (錯誤碼 406):                                                │ │
  │ │ 請登入永豐期貨官網簽署「API電子交易風險預告書暨使用同意書」              │ │
  │ ╰──────────────────────────────────────────────────────────────────────────╯ │
  ```
- **原因說明**：
  這是因為您的永豐金期貨帳戶**尚未簽署電子交易風險預告書**，永豐伺服器限制了該 API 的帳戶讀取權限，回傳 `code: 406, detail: Account Not Acceptable.`。
- **解決方式**：
  請使用電腦或手機登入永豐期貨官方網站的會員中心，於交易權限簽署專區中線上簽署 **「API 電子交易風險預告書暨使用同意書」**。簽署成功約 10-15 分鐘生效後，重新啟動本監控程式，即可順利讀取到您的真實餘額與部位。

## 🪵 日誌與日誌輪轉
- 所有的詳細日誌將寫入專案目錄下的 `app.log`。
- 本程式會於每日午夜進行日誌輪轉（Log Rotation），自動將當日日誌歸檔（例如 `app.log.2026-05-27`），並預設最多保留 **30 天**的歷史紀錄以避免撐爆硬碟空間。
